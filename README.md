# PROTOTYPE/DRAFT Bike Downtown Ottawa. CSAP Commuter Map

A simple standalone map highlighting **car-separated bike routes from Ottawa's
suburbs to downtown**, built for the **Cycling Safety Awareness Program (CSAP)**
administered by [Envirocentre](https://www.envirocentre.ca) on behalf of the
City of Ottawa.

The goal: show people who commute downtown by car that there is a safe,
mostly car-free bike route from their neighbourhood and roughly how long
it takes.

## What's on the map

- **Seven candidate commuter corridors** (Orléans, Blackburn Hamlet, Kanata North, Stittsville,
  Barrhaven, South Keys/Hunt Club, and Nepean), routed along
  the NCC multi-use pathway network wherever possible.
- The **base map** highlights cycling infrastructure: solid teal = Protected bike lane or multi-use-pathway (car-free), dashed teal = Pedestrian priority pathway (car-free), muted =
  On-street bike lane.
- English/French toggle and a high-contrast accessible style variant.

## How it's built

Everything is static, no server, no build step to deploy. Host the folder
anywhere (GitHub Pages works out of the box via the included workflow).

| File | What it is |
|---|---|
| `index.html` | The whole app: MapLibre GL map, route list, legend, i18n |
| `style-default.json`, `style-high-contrast.json` | MapLibre style, generated by `tools/generate-style.mts` |
| `routes.geojson` | The six commuter routes, generated by `tools/fetch-routes.mjs` |
| `vendor/` | MapLibre GL JS 5.19.0, vendored |

### Regenerating the map style

The style is adapted from the excellent open-source
[whereto.bike / bike-app-astro](https://github.com/eljojo/bike-app-astro)
cycling map ("Oasis in the Desert" design). To regenerate, check out that
repo as a sibling folder named `bike-app-astro-main`, then:

```sh
npx tsx tools/generate-style.mts <THUNDERFOREST_API_KEY>
```

Tiles are [Thunderforest outdoors-v2](https://www.thunderforest.com/docs/thunderforest.outdoors-v2/)
vector tiles (OpenStreetMap data). The API key in the style JSON is a
usage-limited free-tier key; regenerate with your own if needed.

### Regenerating the routes

Routes are computed by the public [BRouter](https://brouter.de) instance
using the `trekking` profile, with via-points pinning each corridor to the
intended pathway. Edit the corridor definitions in `tools/fetch-routes.mjs`
and run:

```sh
node tools/fetch-routes.mjs
```
